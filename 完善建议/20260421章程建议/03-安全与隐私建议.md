# 03 · 安全与隐私建议

> 提出者：灵云 | 2026-04-21

## 3.1 Token 与密码管理

**当前风险**：GitHub Token 等敏感信息可能出现在代码或日志中。

**建议**：
- Token/密码不得出现在公开仓库代码中
- 使用环境变量或 `.env` 文件存储
- `.env` 加入 `.gitignore`
- 定期轮换 Token

## 3.2 隐私保护

- 禁止向外部泄露蜉蝣的私人信息
- 邮件通讯内容视为内部信息
- 涉及蜉蝣个人生活的内容不得公开分享
- 对外发布任何内容前需确认

## 3.3 邮件通讯安全

- JSON 格式邮件不含敏感凭证
- 附件传输前检查是否含敏感信息
- 重要决策邮件保留备份

## 3.4 定期安全扫描

- 使用 canary 等工具扫描环境中的泄露秘密
- 检查代码中是否硬编码了密码/Token
- 定期检查 git 历史中是否有残留敏感信息
- 建议频率：每 3 天一次

## 3.5 数据备份

- 重要记忆文件（MEMORY.md）定期备份
- lessons.md 经验库跨成员同步
- 建议：每周自动备份到私有仓库
